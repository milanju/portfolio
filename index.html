Tournament Platform

GitHub View Live

Status: Development

Tools: Meteor, HTML, CSS, JavaScript, Bootstrap, MongoDB, Blaze

This is a real-time tournament platform, specialized to serve StarCraft II tournaments. You could think of this project as an alternative to a tournament platform such as Challonge or Binary Beast, but specialized into serving StarCraft II tournaments, therefore providing a much better user experience.
This website is still under development. The funcionality is fairly complete though. The current demo enables you to create and manage multiple tournaments, let users sign up and once the tournament is started users may report scores of their assigned match. Admins can manipulate the bracket in case of mistakes/false-reports.
Try the demo with following credentials: (USER: admin, PASSWORD: admin).

Barcodes Exposed

GitHub View Live

Status: Live Demo

Tools: HTML, CSS, JavaScript, Material Design Lite, Battle.net game API, Dropbox(backend)

The StarCraft II© grandmaster ladder is dominated by high caliber players hiding their identities, typically using a nick such as lIlIlIlIlIlI. Barcodes Exposed utilizes the Blizzard API to load the current european grandmasters into a table. Finally, known aliases associated with the players unique IDs are gathered from a Dropbox file and displayed next to their account name on the table.
I have chosen the Dropbox backend as a convenient way for multiple people to extend the alias list.
The backend consisting of only the API & a single JSON-File located within a Dropbox folder enables me to host the site on a comfortable GitHub Pages + Dropbox solution.

Tic Tac Toe

GitHub View Live

Status: Live Demo

Tools: Meteor, HTML, CSS, JavaScript, MongoDB, Blaze

Highlight of this little application built on Meteor is the anonymous Match-Making system. The first player to hit the 'Play!' button gets put in a queue. Once another player hits 'Play!' a new board gets created for the two players to play on. Any amount of games may happen at once. I use a heartbeat function to warrant that no disconnected player is sitting in the queue and to clean up inactive boards.
